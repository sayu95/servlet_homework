1. 서블릿 정의	
 서블릿은 클라이언트의 요청을 처리하고 결과를 응답하는 Java 기반의 웹 컴포넌트입니다.

*** 2. 생명주기 ***
* init(): 첫 요청 시 또는 서버 시작 시 단 한 번만 호출. 서블릿 객체를 초기화.
* service(): 요청이 올 때마다 호출. 요청 유형에 따라 doGet()/doPost() 등으로 분기.
* destroy(): 서버 종료 시 단 한 번 호출. 자원 해제. 서블릿은 요청/응답 시 사라지지 않고 메모리에 유지됩니다.

3. doGet/doPost	데이터 위치 차이 (Body 유무)	
doGet은 데이터 노출(URL에 표시), 길이 제한, 북마크 가능의 특징이 있습니다.
doPost는 이와 반대입니다.
4. Request/Response	
요청 정보 / 서버 응답	
보강: Request는 파라미터, 헤더, 세션 정보를 담고, 
Response는 헤더 설정, 콘텐츠 타입 지정, 클라이언트에게 데이터 전송 기능을 제공합니다.

*** 5. 웹 컨테이너 ***	
Java 컴파일, 서블릿과 컨테이너 사이	
정정: 웹 컨테이너는 서블릿의 생명주기 관리 (생성, 초기화, 소멸), 
멀티 스레드 환경 제공, 네트워크 연결 관리를 담당하여 서블릿을 실행 가능하게 하는 실행 환경입니다.

*** 6. Context/Config ***	
Context: Attribute에 저장
Config: 실행 전 초기값 설정

정정:
* Context Init Param: 웹 애플리케이션 전체에서 공유되는 설정 값. (ServletContext 소속)
* Servlet Init Param: 특정 서블릿 한 개만 사용하는 설정 값. (ServletConfig 소속)

문제 번호제시된 답변 (요약)피드백 및 핵심 정정
7. 서블릿 vs JSP서블릿은 웹 컴포넌트, JSP는 템플릿 엔진이다.
보강: JSP는 HTML 내부에 자바 코드를 삽입하여 사용하며, 실행 시 서블릿으로 변환되어 실행됩니다. 
서블릿은 순수 자바 코드로 로직에 집중하며, JSP는 **화면 출력(View)**에 집중합니다.
8. JSP 실행 과정(답변 없음)정답: JSP 파일은 웹 컨테이너에 의해 **서블릿 자바 파일로 변환(Translation)**되고, 컴파일되어 .class 파일이 된 후, 일반 서블릿처럼 실행됩니다.
9. 쿠키 vs 세션저장 공간 차이, 보안/용량 차이보강: 쿠키는 클라이언트(브라우저)에 파일 형태로 저장되며, 세션은 서버의 메모리에 저장됩니다. 세션은 세션 ID만 쿠키를 통해 클라이언트에게 전송하므로 보안이 더 강합니다.
10. 세션 관련 메서드getsessing 이용, invaild 이용정정:  * 세션 객체 획득: request.getSession(true/false) * 데이터 저장/조회: session.setAttribute("key", value) / session.getAttribute("key") * 세션 삭제: session.invalidate() (세션 전체 무효화)
11. MVC 패턴 역할Model: 비즈니스 로직View: 데이터와 결합, 클라이언트에게 보여줌Controller: 요청 전달 및 응답보강: Controller의 핵심 역할은 클라이언트의 요청을 받아(수신), Model에게 일을 시키고, View를 선택하여 응답을 위임하는 중앙 제어 역할입니다.
